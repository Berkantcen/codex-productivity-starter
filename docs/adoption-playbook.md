# Adoption Playbook

Use this playbook when introducing Codex workflows to a team.

## Start With One Workflow

Do not begin with a broad transformation program. Pick one common workflow:

- bug investigation
- small feature implementation
- PR review
- test-gap analysis
- release checklist support

Make that workflow reliable before expanding.

## Pilot Checklist

1. Choose one repository.
2. Add the project starter files.
3. Fill concrete paths and validation commands.
4. Run three real tasks.
5. Compare time spent, review quality, and missed validation.
6. Update the templates based on what Codex had to rediscover.

## Team Guidelines

- Keep humans responsible for product and architectural decisions.
- Keep Codex scoped to the task brief.
- Review generated changes like any other code.
- Treat validation output as part of the deliverable.
- Capture reusable lessons in repo-local files.

## Success Signals

- New tasks need less setup text.
- Final handoffs are easier to review.
- Validation is more consistent.
- Fewer repeated explanations are needed across threads.
- The team knows where to update Codex guidance when practices change.
